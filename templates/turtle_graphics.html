The University Turtle Graphics System

Overview

The GraphicSystem is a Java-based graphical application that builds on the OOPGraphics framework to provide an interactive canvas for drawing and manipulating shapes. The system allows users to issue commands that control a virtual drawing tool, similar to a turtle graphics system, providing functionalities to create various geometric figures and manipulate the drawing environment.

Key Features

  • Graphical User Interface (GUI): The application uses Java's JFrame to create a window where the user can interact with the drawing canvas. This GUI provides real-time visual feedback as users issue commands.
  • Command Processing: The system can process multiple commands through its processCommand method. The available commands include:
    Movement commands: turnleft, turnright, forward, backward
    Shape commands: square, triangle, righttriangle, mandala
    Customization commands: penwidth, pencolour, bgc (background color)
    Utility commands: clear to clear the canvas, save to save the canvas as an image, and load to load an existing image.
  • Saving and Loading: Users can save their current canvas to a file in an image format using the saveImg() method. They can also save drawing commands to a text file for later execution using the toWrite() method. The system offers a prompt to save the canvas before clearing it. The load command allows users to reload previously saved images.
  • Error Handling and User Interaction:The system ensures user interaction through dialog boxes for confirmations (e.g., before clearing the canvas) and prompts for file names when saving the canvas. It includes exception handling for input errors or failed operations, ensuring a smooth user experience.

Technologies Used

  • Programming Language: Java
  • Framework: Java Swing for GUI
  • Libraries: AWT (Abstract Window Toolkit), javax.imageio for image processing
  • Custom Framework: OOPGraphics (extends built-in capabilities for graphical commands)
  • File Handling: BufferedImage, FileWriter, ImageIO for saving/loading images and commands

Challenges & Solutions

A major challenge was implementing the save and load functionality for the drawing canvas. Initially, managing image data and command history in an efficient and user-friendly way was difficult. I overcame this by utilizing `BufferedImage` and `FileWriter` for image saving and command logging, ensuring integration between the graphical interface and file handling.
